How to build wxWidgets from source code on Windows with CMake

Introducion

wxWidgets is a C++ library for cross-platform GUI apps. It calls operating systems's native/choosen controls (Win32, GTK, Qt, Cocoa).
Because wxWidgets depends on GUI libraries which are likely not installed in your computer (development files), the build and installation will not take a short while.
If you stop here after reading the sentence above, install precompiled library here.
Else continue.

Prepare

You will need:

  • C++ compiler
  • CMake and other supported build system (Make, Ninja) since CMake will generate files to one of them
  • Windows SDK, of course.
  • Time
  • Git

It's the best to just install Visual Studio and start Developer Command Prompt.
All external tools must be added to %PATH%.

You also need to set these variables:

  • %WXWIN%: Where to install wxWidgets
  • %WXCFG%: Relative path to build.cfg file (start from %WXWIN%). Use the one matching your target build type (static, shared) and computer architecture.
  • %ARCH%: Target build architecture

Bake the cake!

1. Start (Developer) Command Prompt with all needed tools are in %PATH%.

2. cd to where to clone the project. Clone wxWidgets with --recursive flag.

3. cd to the source code directory.

4. Run:

            md build
            cd build
            cmake -G [generator] .. -DCMAKE_INSTALL_PREFIX=%WXWIN% -DCMAKE_GENERATOR_PLATFORM=%ARCH%
            cmake --build . --target install

That's all! Now wait to the build and installation to finish.
Use wx-config-win to test if your installation works.
Enjoy!
